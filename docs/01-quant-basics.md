# 第一章：量化交易入门

## 本章学习目标

- 理解量化交易的定义，知道它和主观交易有什么本质区别
- 了解量化交易的完整工作流程
- 掌握 A 股市场的基本规则（涨跌停、T+1 等）
- 知道常见量化策略的分类
- 了解 myQuant 项目采用的策略属于哪一类，以及为什么选择它

---

## 1. 什么是量化交易？

**量化交易**（Quantitative Trading），简单说就是：**用数学和计算机程序来做投资决策**。

传统（主观）投资者靠的是：直觉、经验、新闻资讯、K 线"看图"——这些都很难复现和验证。

量化投资者靠的是：历史数据、数学模型、回测验证——规则明确、可重复、可量化。

### 一个具体的对比例子

假设你要决定什么时候买贵州茅台（600519）：

**主观交易者的思路：**
> "最近茅台跌了不少，感觉快到底了，而且过年快到了应该会涨，我买！"

**量化交易者的思路：**
> "过去 5 年的数据显示，当茅台股价低于 250 日均线 8% 以上时，买入后 3 个月的平均收益率是 +12%，胜率 70%，我按照这个规则买入。"

关键区别：量化交易者能告诉你这个决策**在历史上表现如何**，而主观交易者只能凭感觉。

## 2. 量化交易的核心流程

量化交易并不神秘，它的工作流程可以分为四个步骤：

```
┌──────────────┐     ┌──────────────┐     ┌──────────────┐     ┌──────────────┐
│  1. 数据获取  │────▶│ 2. 信号生成  │────▶│ 3. 下单执行  │────▶│ 4. 风险管理  │
│              │     │              │     │              │     │              │
│ 历史K线      │     │ 计算均线     │     │ 买入/卖出    │     │ 止损/仓位    │
│ 实时行情     │     │ 乖离率       │     │ 记录持仓     │     │ 监控回撤     │
│ 财务数据     │     │ 触发阈值？   │     │ 报警通知     │     │ 调整参数     │
└──────────────┘     └──────────────┘     └──────────────┘     └──────────────┘
```

### 第一步：数据获取

万事数据为先。没有数据，就谈不上任何策略。

常见的数据类型：
- **历史 K 线**：每天的开盘价、最高价、最低价、收盘价、成交量（即 OHLCV）
- **实时行情**：当前最新成交价、买卖盘口
- **财务数据**：市盈率、市净率、营收、利润等基本面数据
- **另类数据**：新闻舆情、社交媒体情绪等

myQuant 项目使用的数据：腾讯财经的 K 线数据（免费）

### 第二步：信号生成

这是策略的核心——根据数据计算**买卖信号**。

信号可以基于：
- 技术指标（均线、RSI、MACD 等）
- 基本面指标（市盈率低于某阈值买入）
- 统计规律（价格偏离历史均值太远则均值回归）

myQuant 项目的信号：当 MA250 乖离率 ≤ -8% 时买入，≥ +15% 时卖出。

### 第三步：下单执行

信号出来了，接下来要执行：
- 买多少？（仓位管理）
- 什么价格买？（市价单 or 限价单）
- 实际下单（需要券商 API，国内常用富途、老虎等）

myQuant 目前实现的是**报警功能**（邮件通知），提醒用户手动操作，尚未实现自动下单。

### 第四步：风险管理

这是最容易被忽视、却最重要的一步：
- 这笔交易亏损最多承受多少？（止损）
- 总体上最多亏损多少？（最大回撤控制）
- 要不要分散投资？（不要把鸡蛋放在一个篮子里）

我们会在[第六章：风险管理基础](06-risk-management.md)中详细讲解。

## 3. A 股市场基础知识

如果你完全没有炒股经历，这里介绍一些 A 股的基本规则，这些规则对量化策略设计有直接影响。

### 3.1 交易时间

A 股交易时间（北京时间）：
- **上午**：09:30 ~ 11:30（2 小时）
- **下午**：13:00 ~ 15:00（2 小时）
- 每天共 **4 小时**交易时间
- 法定节假日、周末不交易

> 💡 **对量化的影响**：监控程序需要知道当前是否在交易时间，才能决定是查实时价还是查收盘价。myQuant 使用 `chinese_calendar` 库处理节假日判断。

### 3.2 涨跌停板制度

A 股有**涨跌停**限制，防止股价单日波动过大：
- 普通股票：涨/跌幅不超过 **±10%**
- 科创板、创业板：**±20%**
- ST 股（问题股）：**±5%**

> 💡 **对量化的影响**：如果策略在跌停板触发买入信号，但价格已经跌停无法成交，回测结果会高估实际收益。

### 3.3 T+1 制度

A 股实行 **T+1 交易**：今天买入的股票，**明天才能卖出**。

> 💡 **对量化的影响**：策略不能在同一天买入后立即卖出。myQuant 的回测引擎使用日线数据，默认每天最多一次操作，规避了这个问题。

### 3.4 个股 vs ETF vs 场外基金

这三类标的有重要区别，myQuant 全部支持：

| 类型 | 代码示例 | 交易方式 | 最小单位 | 特点 |
|------|---------|---------|---------|------|
| **个股** | 600519（贵州茅台） | 竞价交易 | 100 股（1 手） | 波动大，有涨跌停 |
| **ETF（场内基金）** | 510300（沪深300ETF） | 竞价交易（同个股） | 100 份（1 手） | 分散投资，波动小 |
| **场外基金** | 012708（某债基） | 申购/赎回（按净值） | 通常 0.01 元起 | T+1 或 T+2 确认净值，无法实时交易 |

> A 股一共约 5000 支个股，上交所（sh）代码以 6/5/688 开头，深交所（sz）代码以 0/3 开头，北交所（bj）以 8/4 开头。

### 3.5 什么是均线？什么是年线？

A 股市场中有一个约定俗成的"年线"概念：

- A 股每年大约有 **250 个交易日**（去掉节假日后）
- **MA250**（250 日均线）= 过去 250 个交易日的收盘价均值
- 因为覆盖了大约一整年，所以 MA250 被称为**年线**

年线是许多量化策略和主观交易者都非常重视的一条均线，被认为代表了股票的"合理价值中枢"。

## 4. 常见量化策略分类

量化策略有很多种，这里介绍四种最常见的类型：

### 4.1 趋势跟踪策略

**核心思路**：涨的继续买，跌的继续卖。

> 例子：当股价从下方突破 MA20（20 日均线）时买入，跌破时卖出。

**优点**：能抓住大趋势行情  
**缺点**：震荡市频繁止损，磨耗严重

适合行情：单边上涨或单边下跌的趋势市场

### 4.2 均值回归策略（myQuant 采用）

**核心思路**：价格不可能永远偏离均值，偏得越远越可能回来。

> 例子：当股价跌到年线以下 8% 时，认为它"跌多了"，买入等待回归。
> 当股价涨到年线以上 15% 时，认为它"涨多了"，卖出锁定利润。

**优点**：逻辑简单，适合震荡市  
**缺点**：遇到单边下跌（如熊市）可能越买越跌，"接飞刀"风险

适合行情：震荡盘整，没有持续单边趋势的市场

### 4.3 套利策略

**核心思路**：利用同一资产在不同市场（或形式）之间的价差盈利。

> 例子：某 ETF 的市价比其净值低 0.5%，买入 ETF 同时卖出成份股组合，锁定利润。

**优点**：理论上低风险  
**缺点**：需要复杂的交易基础设施，窗口期极短，机构才能真正做到

### 4.4 因子选股策略

**核心思路**：通过大量数据研究，找到与未来股价涨跌相关的"因子"（特征），按因子排名选股。

> 例子：低市盈率因子（PE 越低，预期收益越高）、低波动因子、动量因子等。

**优点**：学术研究充分，有理论依据  
**缺点**：需要大量数据处理，单因子效果在中国市场持续衰减

## 5. myQuant 采用的策略：均值回归

### 为什么选择均值回归策略？

myQuant 采用的是**MA250 乖离率均值回归策略**。

选择这个策略的原因：

1. **逻辑直观**：价格偏离年线太远，有道理认为会回归
2. **计算简单**：只需要一条均线和一个比值，初学者容易理解
3. **参数少**：只有买入阈值和卖出阈值两个参数，不容易过拟合
4. **适合 A 股**：A 股散户多，情绪驱动明显，短期过度反应（超跌/超涨）后回归均值的现象相对常见
5. **易于回测验证**：历史数据充足，可以在不同股票、不同时期验证策略稳定性

### 策略的局限性

当然，没有"完美"的策略。均值回归策略的主要风险：

- **熊市接飞刀**：2015、2018 年等大熊市中，价格可以长期持续偏离均线向下，越买越亏
- **基本面风险**：某些个股因经营恶化而持续下跌，它可能永远不会"回归"均值
- **全仓操作风险**：myQuant 目前采用全仓买卖，单次错误的代价较大

我们会在[第三章：MA250 乖离率策略深度解析](03-bias-strategy.md)和[第六章：风险管理基础](06-risk-management.md)中详细讨论这些问题。
