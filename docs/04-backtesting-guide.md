# 第四章：回测入门与实践

## 本章学习目标

- 理解回测的含义和意义
- 掌握回测的基本流程
- 理解并会计算总收益率、年化收益率、最大回撤、胜率等关键指标
- 认识回测的常见陷阱（过拟合、幸存者偏差等）
- 能够使用 myQuant 的回测接口并正确解读结果

## 1. 什么是回测？

### 1.1 定义

**回测（Backtesting）**就是把一个交易策略放到**历史数据上**，模拟它在过去的表现，评估策略是否有效。

类比一下：在学开车之前，你可以在模拟器上练习，看看你的驾驶风格在各种路况下的表现——回测就是量化策略的"模拟驾驶"。

### 1.2 为什么要回测？

1. **验证假设**：你认为"跌破年线 8% 时买入"是好策略，但历史上真的赚钱吗？
2. **量化风险**：最坏情况下会亏多少？（最大回撤）
3. **优化参数**：-8% 买入还是 -10% 买入更好？不同参数差别多大？
4. **建立信心**：看到历史验证的结果，实盘时才能坚持执行策略
5. **发现问题**：策略可能在某些市场环境下效果很差，回测可以提前发现

### 1.3 回测 ≠ 未来预测

⚠️ **最重要的提醒**：回测优秀不代表未来一定盈利！

原因：市场在变，历史规律未必持续。回测只是评估工具，不是水晶球。

## 2. 回测的基本流程

myQuant 的回测引擎（`backtester.py`）按以下流程运行：

```
1. 获取历史 K 线数据
        │
        ▼
2. 过滤指定日期范围
        │
        ▼
3. 从第 250 条数据开始（确保 MA250 有效）
        │
        ▼
4. 逐日模拟：
   每一天 → 计算 MA250 乖离率 → 判断信号 → 模拟买入/卖出 → 记录资产
        │
        ▼
5. 汇总指标：
   总收益率、年化收益率、最大回撤、胜率、交易次数…
        │
        ▼
6. 返回结果（含每笔交易记录和资金曲线）
```

## 3. 关键评估指标详解

### 3.1 总收益率

$$\text{总收益率} = \frac{\text{最终资产} - \text{初始资金}}{\text{初始资金}}$$

**例子**：
- 初始资金：100,000 元
- 回测结束时资产：132,560 元
- 总收益率 = (132,560 - 100,000) / 100,000 = **+32.56%**

### 3.2 年化收益率

总收益率不方便横向对比（持有 1 年 +30% vs 持有 3 年 +30%，哪个更好？），因此需要换算成年化。

**计算公式（复利）：**

$$\text{年化收益率} = \left(1 + \text{总收益率}\right)^{\frac{365}{\text{持有天数}}} - 1$$

**例子**：
- 总收益率 +32.56%，持有天数 1461 天（约 4 年）

$$\text{年化} = (1 + 0.3256)^{\frac{365}{1461}} - 1 = 1.3256^{0.2497} - 1 \approx 7.41\%$$

解读：相当于每年赚 7.41%，考虑到策略的风险，这是一个需要与基准收益对比才能判断好坏的数字。

### 3.3 最大回撤（Max Drawdown）

最大回撤是衡量风险的核心指标，描述在某段时间内，**资产从峰值到谷底的最大跌幅**。

**计算方法：**

$$\text{最大回撤} = \frac{\text{某历史谷底} - \text{之前的历史峰值}}{\text{之前的历史峰值}}$$

取所有这样的比值中**绝对值最大**的那个（最大回撤是负数）。

**图示：**

```
资产
 │
140k┤          ●  ← 峰值 (Peak)
    │        ╱  ╲
130k┤      ╱      ╲
    │    ╱          ╲
120k┤  ╱              ● ← 谷底 (Trough)
    │╱
110k┤
    │
100k┤● 初始
    └──────────────────── 时间
```

$$\text{最大回撤} = \frac{120k - 140k}{140k} = -14.3\%$$

**为什么最大回撤重要？**

想象你的策略历史总收益是 +80%，但中途经历了一次 -60% 的回撤——大多数人在亏损 -30% 时就已经心态崩溃、割肉离场了。

> **实战原则**：最大回撤超过 -30%，大多数人就无法坚持持有了。选择策略时，最大回撤是比收益率更重要的指标。

**myQuant 中的实现：**

```python
# 来自 app/services/backtester.py
def _calc_max_drawdown(equity_curve: List[float]) -> float:
    """计算最大回撤（返回负数）。"""
    if not equity_curve:
        return 0.0
    peak = equity_curve[0]   # 当前历史最高点
    max_dd = 0.0
    for v in equity_curve:
        if v > peak:
            peak = v          # 更新峰值
        dd = (v - peak) / peak if peak > 0 else 0.0
        if dd < max_dd:
            max_dd = dd       # 记录最大跌幅
    return max_dd
```

### 3.4 胜率

$$\text{胜率} = \frac{\text{盈利卖出次数}}{\text{总卖出次数}}$$

**例子**：一共卖出 4 次，其中 3 次卖出价 > 买入价，胜率 = 3/4 = 75%。

**注意**：高胜率不等于高收益！

| 策略 | 胜率 | 平均盈利 | 平均亏损 | 期望收益 |
|------|------|---------|---------|---------|
| A | 70% | +5% | -20% | 70%×5% + 30%×(-20%) = **-2.5%** |
| B | 40% | +30% | -5% | 40%×30% + 60%×(-5%) = **+9%** |

策略 B 胜率更低，但期望收益更高！这就是**盈亏比**的重要性。

### 3.5 基准收益与超额收益

**基准收益（Benchmark Return）**：
用同样的初始资金，在回测起始日买入并持有到结束日的收益率（即"买入持有"策略）。

$$\text{基准收益} = \frac{\text{结束日价格} - \text{起始日价格}}{\text{起始日价格}}$$

**超额收益（Alpha）**：

$$\text{超额收益} = \text{策略总收益} - \text{基准收益}$$

**解读**：如果策略总收益是 +32%，基准收益是 +50%，那超额收益是 **-18%**——策略跑输了"无脑持有"，这个策略是失败的！

> 💡 **核心评判标准**：策略的收益 **必须** 超越基准，否则直接买入持有就好，为什么要复杂的策略？

## 4. 回测的常见陷阱

### 4.1 过拟合（最危险的陷阱）

**什么是过拟合？**

就是把策略参数调整得在历史数据上"完美"，但实际上只是记住了历史，无法泛化到未来。

**一个极端例子**：
- 你测试了 1000 种参数组合
- 某组合在过去 10 年的历史数据上，总收益恰好是最高的
- 但这不是因为这组参数真的好，而是"碰巧"在历史数据上最优
- 换到未来 1 年，它可能是表现最差的

**避免过拟合的方法**：
1. 参数尽量少（MA250 乖离率策略只有 2 个参数，相对安全）
2. 在历史数据外留一段"测试集"，不参与参数优化
3. 在多只不同股票上验证策略，而不是只在一只股票上调参
4. 策略逻辑要有经济学意义（不能只是数字凑巧）

### 4.2 幸存者偏差

**什么是幸存者偏差？**

你回测的股票池，都是"活下来"的股票——因为已经退市的股票不在数据库里了。

用当前存活的股票做回测，会高估策略收益：那些因为经营失败而持续下跌退市的股票，均值回归策略在它们身上会失败，但你根本看不到这些失败案例。

**解决方法**：
- 尽量用宽基指数/ETF（不会退市）
- 如果用个股，选择历史悠久、基本面稳健的大蓝筹

### 4.3 未来函数（Look-ahead Bias）

**什么是未来函数？**

在计算信号时，不小心用到了"未来才能知道"的数据。

**例子**：
- 在某天的收盘时刻，用"当天的最高价"来触发买入信号
- 但实际上你在当天开盘时根本不知道最高价是多少（只有收盘后才知道）

myQuant 的回测引擎使用**收盘价**计算 MA250，并且每天只用当天收盘后已知的数据，没有未来函数问题。

### 4.4 忽略交易成本

实际交易有手续费（通常 0.03%~0.1%），买卖一次来回成本约 0.1%~0.2%。

对于交易频率很高的策略，这个成本会严重侵蚀收益。myQuant 的回测目前**没有计入手续费**，实盘时要相应折扣预期收益。

## 5. 如何使用 myQuant 的回测接口

### 5.1 发起回测请求

```bash
curl -X POST http://localhost:8000/backtest/single \
  -H "Content-Type: application/json" \
  -d '{
    "code": "510300",
    "buy_bias_rate": -0.05,
    "sell_bias_rate": 0.10,
    "initial_capital": 100000,
    "start_date": "2018-01-01",
    "end_date": "2024-01-01"
  }'
```

**参数说明：**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `code` | string | ✅ | 证券代码，如 `510300`（沪深300ETF）|
| `buy_bias_rate` | float | ✅ | 买入乖离率阈值，必须为负数，如 `-0.05` |
| `sell_bias_rate` | float | ✅ | 卖出乖离率阈值，必须为正数，如 `0.10` |
| `initial_capital` | float | ❌ | 初始资金（元），默认 100000 |
| `start_date` | string | ❌ | 回测起始日（YYYY-MM-DD），默认数据最早日期 |
| `end_date` | string | ❌ | 回测结束日（YYYY-MM-DD），默认今天 |

> 💡 回测接口不需要预先通过 `POST /targets/` 添加标的，直接传入证券代码即可，系统会自动解析证券名称和类型。

### 5.2 解读回测结果

一个完整的响应结构：

```json
{
  "code": "510300",
  "name": "沪深300ETF",
  "type": "etf",
  "period": {
    "start": "2018-01-02",    // 实际回测起始日
    "end": "2023-12-29"       // 实际回测结束日
  },
  "params": {
    "buy_bias_rate": -0.05,   // 你设置的买入阈值
    "sell_bias_rate": 0.10,   // 你设置的卖出阈值
    "initial_capital": 100000.0
  },
  "summary": {
    "total_return": 0.2856,            // 总收益率 +28.56%
    "total_return_pct": "28.56%",
    "annualized_return": 0.0432,       // 年化收益率 4.32%
    "annualized_return_pct": "4.32%",
    "max_drawdown": -0.1821,           // 最大回撤 -18.21%
    "max_drawdown_pct": "-18.21%",
    "trade_count": 12,                 // 总交易次数（买+卖=6笔往返）
    "win_rate": 0.6667,                // 胜率 66.67%
    "win_rate_pct": "66.67%",
    "final_capital": 128560.0,         // 最终资产
    "benchmark_return": 0.1500,        // 基准收益（买入持有）+15%
    "benchmark_return_pct": "15.00%",
    "excess_return": 0.1356,           // 超额收益 +13.56%
    "excess_return_pct": "13.56%"
  },
  "trades": [...]   // 每笔交易详情
}
```

**如何判断结果好不好？**

| 指标 | 判断标准 |
|------|---------|
| 年化收益率 | 需要超越基准（或同期余额宝/债券利率） |
| 最大回撤 | 越小越好；一般要求 < -20%，理想 < -15% |
| 胜率 | 高胜率好，但要结合盈亏比看 |
| 超额收益 | 必须为正，否则不如直接买入持有 |
| 交易次数 | 太少（< 5 次）说明信号太稀，统计意义弱 |

## 6. 不同参数下的回测对比建议

### 6.1 系统性参数扫描

建议用不同的参数组合分别回测，找到相对稳健的参数：

```bash
# 买入阈值：-5%, -8%, -10%, -12%
# 卖出阈值：+8%, +10%, +15%, +20%

# 例如测试 4×4 = 16 种组合，比较汇总指标
```

### 6.2 示例对比表

| buy_bias | sell_bias | 年化收益 | 最大回撤 | 交易次数 | 超额收益 |
|---------|---------|---------|---------|---------|---------|
| -5% | +10% | 6.2% | -22% | 14 | +3.1% |
| -8% | +15% | 4.3% | -18% | 6 | +1.2% |
| -10% | +15% | 5.1% | -16% | 4 | +2.0% |
| -12% | +20% | 3.8% | -12% | 2 | +0.7% |

> 以上数据为虚构示例，仅展示分析思路，实际结果需自行回测。

**观察规律**：
- 阈值越宽松（绝对值小），交易次数越多，但每次进入不够"超跌"，胜率可能低
- 阈值越严格（绝对值大），交易次数少，但每次进入时已经跌得很深，风险也更大

**最终建议**：选择在多种参数下都表现稳定（而不是某套参数极端优秀）的区间，避免过拟合。
