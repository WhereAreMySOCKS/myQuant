## 1. 策略核心思想：均值回归

### 1.1 什么是均值回归？

均值回归（Mean Reversion）是一个统计学概念，描述的是：**偏离均值越远的事物，越有向均值靠拢的倾向**。

在金融市场中，这个现象也大量存在，原因包括：
1. **套利机制**：股价跌得"离谱"便宜时，价值投资者会买入，价格自然回升
2. **情绪修正**：市场恐慌引发的超卖，随着情绪平息会自然修复
3. **基本面锚定**：企业的内在价值（盈利、资产）是股价的"锚"，价格偏离太远会被拉回

### 1.2 一个生活中的例子

想象天气温度（类比股价）围绕一个"正常温度"（类比 MA250）波动：
- 某天气温突然降到 -20℃（极端超跌）→ 大概率过几天会回暖
- 某天气温突然升到 50℃（极端超涨）→ 大概率过几天会降温

当然，气候会变化（类比股票基本面变化），所以这个比喻并不完美——但直觉上是成立的。

### 1.3 为什么 MA250 是好的"均值"？

使用 MA250 作为均值参考而不是简单地用历史平均价，原因：
- MA250 是**滚动计算**的，能随时间自适应（比如股票整体上涨后，MA250 也会跟着上移）
- 250 天 ≈ 一整个年度，包含了完整的季度周期、分红周期
- 被大量投资者"自我实现"地关注，成为重要的心理价位

## 2. 策略的买入逻辑

### 2.1 买入条件

$$\text{乖离率} = \frac{\text{当前价} - \text{MA250}}{\text{MA250}} \leq \text{buy\_bias\_rate}$$

其中 `buy_bias_rate` 是一个**负数**，代表下行偏离阈值。

**例子**：`buy_bias_rate = -0.08`（即 -8%）

- 贵州茅台当前价 **1600 元**，MA250 = **1750 元**
- 乖离率 = (1600 - 1750) / 1750 ≈ **-0.0857（-8.57%）**
- -8.57% ≤ -8%？**是的** → 触发 **BUY 信号** ✅

- 贵州茅台当前价 **1640 元**，MA250 = **1750 元**
- 乖离率 = (1640 - 1750) / 1750 ≈ **-0.0629（-6.29%）**
- -6.29% ≤ -8%？**否** → 不触发买入信号 ❌

### 2.2 买入的经济含义

触发买入时，意味着：
- 股价比过去一年的平均价格**便宜了至少 8%**
- 市场短期情绪可能过于悲观
- 从均值回归角度看，有一定的"安全边际"

### 2.3 代码实现

```python
# 来自 app/services/backtester.py（主循环）
bias_rate = (price - ma250) / ma250

# 买入：空仓状态 + 乖离率低于买入阈值
elif shares == 0 and bias_rate <= buy_bias_rate:
    # 全仓买入，按 100 股最小手数取整
    max_shares = int(cash / price / _LOT_SIZE) * _LOT_SIZE  # _LOT_SIZE = 100
    if max_shares > 0:
        amount = max_shares * price
        cash -= amount
        shares = max_shares
        last_buy_price = price
```

注意：
- `shares == 0`：只有空仓时才买入（不加仓）
- `int(cash / price / 100) * 100`：按 100 股（1 手）取整，符合 A 股最小交易单位

## 3. 策略的卖出逻辑

### 3.1 卖出条件

$$\text{乖离率} = \frac{\text{当前价} - \text{MA250}}{\text{MA250}} \geq \text{sell\_bias\_rate}$$

其中 `sell_bias_rate` 是一个**正数**，代表上行偏离阈值。

**例子**：`sell_bias_rate = 0.15`（即 +15%）

- 贵州茅台当前价 **2030 元**，MA250 = **1750 元**
- 乖离率 = (2030 - 1750) / 1750 ≈ **+0.16（+16%）**
- +16% ≥ +15%？**是的** → 触发 **SELL 信号** ✅

- 贵州茅台当前价 **1950 元**，MA250 = **1750 元**
- 乖离率 = (1950 - 1750) / 1750 ≈ **+0.114（+11.4%）**
- +11.4% ≥ +15%？**否** → 不触发卖出信号 ❌

### 3.2 卖出的经济含义

触发卖出时，意味着：
- 股价比过去一年的平均价格**贵了至少 15%**
- 市场短期情绪可能过于乐观
- 从均值回归角度看，继续上涨的空间已经不大，风险在积累

### 3.3 卖出优先原则

```python
# 来自 app/services/backtester.py
# 卖出优先（保守策略）
if shares > 0 and bias_rate >= sell_bias_rate:
    # 先检查是否卖出
    ...
elif shares == 0 and bias_rate <= buy_bias_rate:
    # 再检查是否买入
    ...
```

代码中卖出条件的检查在买入之前。这是"保守策略"：当买卖信号同时满足（理论上不太可能，但防御性编程）时，优先卖出止盈。

### 3.4 信号判断（监控模式）

在实时监控中，信号判断逻辑在 `analyzer.py` 的 `check_signal` 函数中：

```python
# 来自 app/services/analyzer.py
def check_signal(indicators, buy_bias_rate=None, sell_bias_rate=None, ...):
    # 卖出优先
    if sell_bias_rate is not None:
        if indicators['bias_rate'] >= sell_bias_rate:
            return 'SELL'

    # 买入
    if buy_bias_rate is not None:
        if indicators['bias_rate'] <= buy_bias_rate:
            return 'BUY'

    return None
```

## 4. 参数含义和调优建议

### 4.1 两个核心参数

| 参数 | 说明 | 推荐范围 | 示例值 |
|------|------|---------|--------|
| `buy_bias_rate` | 买入乖离率阈值（负数） | -5% ~ -15% | -0.08 (-8%) |
| `sell_bias_rate` | 卖出乖离率阈值（正数） | +8% ~ +25% | 0.15 (+15%) |

### 4.2 参数如何影响策略行为

**买入阈值 `buy_bias_rate` 越负（绝对值越大）：**
- 买入条件越苛刻，触发越少
- 每次买入时已经跌得更深，安全边际更高
- 但可能长期无法买入（等不到那么低的价格）

**卖出阈值 `sell_bias_rate` 越大：**
- 卖出条件越苛刻，让利润跑得更远
- 但也可能利润回吐（涨了 20% 再跌回 15% 才卖）

### 4.3 不同标的的参数建议

不同类型的标的，波动率不同，阈值要相应调整：

| 标的类型 | 波动特性 | 建议买入阈值 | 建议卖出阈值 |
|---------|---------|------------|------------|
| 宽基 ETF（如沪深300 ETF） | 低波动 | -5% ~ -8% | +8% ~ +12% |
| 蓝筹个股（如贵州茅台） | 中等波动 | -8% ~ -12% | +12% ~ +20% |
| 中小盘成长股 | 高波动 | -12% ~ -20% | +20% ~ +35% |
| 行业 ETF | 中高波动 | -8% ~ -15% | +15% ~ +25% |

> ⚠️ **重要提醒**：上表仅供参考，最终应通过**历史回测**确定每只标的的最优参数，而不是凭感觉设置！详见[第四章：回测入门与实践](04-backtesting-guide.md)。

### 4.4 参数调优的基本思路

1. 先用合理的初始值（如 buy=-0.08, sell=0.15）进行回测
2. 观察回测结果：交易次数太少说明阈值太严，太多说明太松
3. 在合理范围内调整参数，比较回测结果
4. 注意：不要仅以"总收益率最高"为目标，还要看最大回撤和交易次数

## 5. 全仓 vs 分仓策略

### 5.1 myQuant 当前策略：全仓

myQuant 目前采用**全仓操作**：
- 买入时用**全部可用资金**买入（按手取整）
- 卖出时**清空全部持仓**

```python
# 全仓买入
max_shares = int(cash / price / _LOT_SIZE) * _LOT_SIZE  # 用所有资金买入最多整手数
```

**优点：**
- 逻辑简单，回测容易实现
- 每次都是"信心满满"的入场

**缺点：**
- 买错一次，损失惨重（没有平均摊薄的机会）
- 如果信号后价格继续下跌，无法加仓摊薄成本

### 5.2 更稳健的做法：分批建仓

实际操作中，许多量化交易者会采用**分批买入**的方式：

| 乖离率触达 | 操作 | 仓位 |
|----------|------|------|
| -5% | 第一批买入 | 1/3 仓 |
| -8% | 第二批买入 | 再加 1/3 仓 |
| -12% | 第三批买入 | 再加 1/3 仓 |
| +15% | 清仓卖出 | 0 仓 |

这样的"金字塔加仓"能：
- 降低平均持仓成本
- 控制每次错误买入的风险
- 在连续下跌时仍有子弹可用

> myQuant 后续版本计划支持分批建仓配置。

## 6. 策略的优势

1. **信号明确，执行纪律强**：有明确的数学条件，不受情绪影响
2. **逻辑简单，不易过拟合**：只有两个参数，历史验证充分
3. **适合宽基指数/ETF**：指数不会退市，均值回归的前提更加成立
4. **降低追涨杀跌的概率**：在价格低时买入，高时卖出，天然逆势操作

---

## 7. 策略的局限性

### 7.1 不适合单边趋势市场

均值回归策略在**震荡市**中表现最好，在**单边趋势市**中会失效：

- **熊市**：价格可以长期持续低于年线，每次"超跌买入"都会继续亏损（接飞刀）
- **牛市**：价格可以长期持续高于年线，"超涨卖出"会过早止盈，错失更大涨幅

**历史案例**：
- 2015 年 A 股大熊市：股价在年线以下持续下跌超过 50%
- 2020 年疫情后美股快速反弹：如果过早以"超涨"理由卖出，会错失大量收益

### 7.2 个股退市风险

均值回归假设价格最终会回到均线。但如果一家公司基本面严重恶化：
- 股价可能永久性下跌，永远不回归
- 这就是为什么**宽基 ETF** 比单只个股更适合这个策略

> ⚠️ **强烈建议**：初学者优先用宽基 ETF（沪深300、中证500等）测试策略，而不是个别股票

### 7.3 参数敏感性

不同的阈值组合，回测结果差异可能很大。如果过度优化参数（在历史数据上找到"最优"组合），未来实盘效果可能大幅下降，这就是**过拟合**问题。

更多关于过拟合的讨论见[第四章：回测入门与实践](04-backtesting-guide.md)。

## 8. 适合的市场环境总结

| 市场环境 | 策略表现 | 建议 |
|---------|---------|------|
| 震荡盘整（大多数时候） | ✅ 表现较好 | 适合使用 |
| 单边上涨（牛市初中期） | ⚠️ 会过早卖出 | 提高卖出阈值 |
| 单边下跌（熊市） | ❌ 持续亏损风险 | 设置止损，控制仓位 |
| 急跌后快速反弹（V型反转） | ✅ 表现极佳 | 策略最理想场景 |

## 9. 完整策略流程图

```
每个交易日检查
      │
      ▼
计算 MA250 乖离率
      │
      ├── 乖离率 >= sell_bias_rate (+15%)？
      │         │
      │         ├── 是，且持有股票 → SELL（全仓卖出）→ 记录交易 → 更新资金
      │         └── 否，继续
      │
      ├── 乖离率 <= buy_bias_rate (-8%)？
      │         │
      │         ├── 是，且空仓 → BUY（全仓买入）→ 记录交易 → 更新持仓
      │         └── 否，继续
      │
      └── 无操作，记录当日资产（cash + 持仓市值）
```
