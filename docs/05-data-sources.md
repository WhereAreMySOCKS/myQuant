## 1. A 股常用免费数据源

很多初学者会问：量化交易需要买数据吗？

好消息是，**A 股有很多免费的公开数据源**，足以支撑基础的量化策略：

| 数据源 | 数据类型 | 特点 | 限制 |
|--------|---------|------|------|
| **腾讯财经** (`qt.gtimg.cn`) | 实时行情、历史K线 | 稳定、快速、无需登录 | 有频率限制 |
| **新浪财经** | 历史K线、实时行情 | 数据完整 | 偶发不稳定 |
| **天天基金** (`fundgz.1234567.com.cn`) | 场外基金实时估值 | 专业基金数据 | 仅场外基金 |
| **东方财富** (`api.fund.eastmoney.com`) | 场外基金历史净值 | 数据全面 | 有反爬机制 |
| **akshare**（Python库） | 覆盖所有类型 | 开源，数据源丰富 | 部分接口不稳定 |
| **tushare**（Python库） | 股票/期货/指数 | 数据质量高 | 免费积分有限 |

### 免费 vs 付费数据

| 维度 | 免费数据 | 付费数据 |
|------|---------|---------|
| 实时性 | 15分钟延迟（部分）或实时 | 实时 Tick 数据 |
| 历史深度 | 通常 1~3 年 | 10~20 年 |
| 稳定性 | 可能封 IP，偶发故障 | SLA 保障 |
| 数据类型 | 行情数据为主 | 财务、另类数据 |
| 适合场景 | 个人学习、简单策略 | 专业机构、高频交易 |

> 对于 myQuant 这样的个人量化项目，免费数据源完全够用。

## 2. myQuant 使用的数据源

### 2.1 数据源配置

```
个股/ETF 实时行情：腾讯财经 qt.gtimg.cn （主）
个股/ETF 历史K线：腾讯财经 web.ifzq.gtimg.cn（主）→ 新浪财经（备）
场外基金实时估值：天天基金 fundgz.1234567.com.cn（主）
场外基金历史净值：东方财富 api.fund.eastmoney.com
```

### 2.2 腾讯财经实时行情接口

```
URL：https://qt.gtimg.cn/q={prefix}{code}
```

其中 `prefix` 根据证券代码自动判断交易所：
- `sh`：上海证券交易所（代码以 6/5 开头）
- `sz`：深圳证券交易所（代码以 0/3 开头）
- `bj`：北京证券交易所（代码以 8/4 开头）

**例子**：查询贵州茅台（600519，上交所）

```
https://qt.gtimg.cn/q=sh600519
```

**返回格式**（原始文本）：
```
v_sh600519="51~贵州茅台~600519~1680.5~1692.8~...";
```

这是一个分隔符为 `~` 的字符串，需要解析各字段。

**代码中的交易所前缀判断（`data_fetcher.py`）：**

```python
def _exchange_prefix(code: str) -> str:
    """根据证券代码判断交易所前缀。"""
    if not code:
        return "sz"
    first = code[0]
    if first in ("6", "5"):
        return "sh"   # 上交所
    if first in ("8", "4"):
        return "bj"   # 北交所
    return "sz"       # 深交所（默认）
```

### 2.3 腾讯财经历史 K 线接口

```
URL：https://web.ifzq.gtimg.cn/appstock/app/fqkline/get
参数：
  param={prefix}{code},day,起始日期,结束日期,数据条数,qfq（前复权）
```

返回 JSON 格式的日线 K 线数据，包含：日期、开盘、收盘、最高、最低、成交量。

### 2.4 天天基金场外基金估值

```
URL：https://fundgz.1234567.com.cn/js/{code}.js
```

这个接口返回场外基金的**实时估算净值**（交易时间内每 15 分钟更新一次），字段包括：
- `gsz`：估算净值
- `gszzl`：估算增长率（%）
- `gztime`：估算时间

## 3. 主备切换机制

### 3.1 为什么需要主备切换？

单一数据源会有以下问题：
- 服务器临时故障
- 接口格式突然变化
- 被反爬机制封锁 IP
- 频率限制触发 429 错误

myQuant 通过"主数据源失败 → 自动切换备用数据源"来提高可用性。

### 3.2 切换逻辑示意

```python
# 伪代码示意（来自 data_fetcher.py 的设计思路）
def fetch_stock_history(code: str):
    # 1. 尝试腾讯财经（主）
    df = _fetch_from_tencent(code)
    if df is not None and not df.empty:
        _mark_source_ok("tencent_history")
        return df

    # 2. 主数据源失败，切换到新浪财经（备）
    _mark_source_fail("tencent_history")
    logger.warning(f"腾讯历史K线失败，切换到新浪: {code}")
    df = _fetch_from_sina(code)
    if df is not None and not df.empty:
        _mark_source_ok("sina_history")
        return df

    # 3. 两个数据源都失败，返回 None
    _mark_source_fail("sina_history")
    logger.error(f"所有历史K线数据源均不可用: {code}")
    return None
```

### 3.3 数据源健康状态监控

`data_fetcher.py` 维护了一个全局的数据源状态字典：

```python
_DATASOURCE_STATUS: dict = {
    "tencent_realtime": {"last_success": None, "last_failure": None},
    "tencent_history":  {"last_success": None, "last_failure": None},
    "sina_history":     {"last_success": None, "last_failure": None},
    "tiantian_estimation": {"last_success": None, "last_failure": None},
    "tiantian_history_nav": {"last_success": None, "last_failure": None},
}
```

每次请求成功/失败都会更新这个状态，方便排查问题。

## 4. 数据字段含义

### 4.1 实时行情字段

| 字段 | 含义 | 例子 |
|------|------|------|
| `price` | 最新成交价（当前价） | 1680.5 |
| `change` | 涨跌额（当前价 - 昨收价） | -12.3 |
| `change_pct` | 涨跌幅 (%) | -0.73 |
| `volume` | 成交量（股数或手数） | 123456（手） |
| `amount` | 成交额（元） | 2.07亿元 |
| `bid1` ~ `bid5` | 买一到买五价格 | 买盘 |
| `ask1` ~ `ask5` | 卖一到卖五价格 | 卖盘 |

### 4.2 历史 K 线字段

| 字段 | 含义 | 英文名 |
|------|------|--------|
| `日期` | 交易日期 | Date |
| `开盘` | 开盘价 | Open |
| `收盘` | 收盘价 | Close |
| `最高` | 最高价 | High |
| `最低` | 最低价 | Low |
| `成交量` | 成交量（股数） | Volume |

myQuant 中主要使用**收盘价（`df['收盘']`）**来计算均线和乖离率。

### 4.3 涨跌幅的计算

$$\text{涨跌幅} = \frac{\text{当前价} - \text{昨日收盘价}}{\text{昨日收盘价}} \times 100\%$$

## 5. 前复权 vs 后复权 vs 不复权

### 5.1 为什么有复权问题？

A 股上市公司经常进行**分红派息**和**送配股**，这些操作会导致股价"突然跳变"，影响历史数据的连续性。

**例子**：某股票昨天收盘价 10 元，今天除息分红 0.5 元/股，今天"除权"后开盘价就变成了 9.5 元。

从 K 线图上看，这就产生了一个 -5% 的"向下跳空"——但实际上持股者并没有亏损，只是收到了 0.5 元现金。

如果不处理这个问题，计算均线时就会产生严重偏差。

### 5.2 三种复权方式

| 复权方式 | 含义 | 适用场景 |
|---------|------|---------|
| **不复权** | 原始历史价格，含分红跳空 | 查看当时实际股价 |
| **前复权** | 以当前价格为基准，将历史价格向上调整 | 计算技术指标、回测（**最常用**） |
| **后复权** | 以上市初始价格为基准，将历史价格向下调整 | 计算长期持有总收益 |

### 5.3 直观对比

假设某股票历史上经历了一次除权（股价从 10 元变成 9 元，分了 1 元现金）：

```
时间轴：  Day1    Day2    Day3（除权日）  Day4
不复权：  10.0    10.2    9.0（跳空）     9.3
前复权：  9.0     9.18    9.0            9.3  ← 历史价格被调低
后复权：  10.0    10.2    10.11          10.44 ← 分红折算到价格里
```

### 5.4 myQuant 使用前复权

```python
# 来自 data_fetcher.py，腾讯历史K线请求参数
param = f"{prefix}{code},day,{start},{end},{count},qfq"
#                                                    ↑
#                          qfq = 前复权（Qian Fu Quan）
```

**选前复权的原因**：
- 计算均线时，需要历史价格和当前价格在同一个坐标系下
- 前复权保证了"当前价格不变，历史价格向下调整"，与当前实时价格直接可比
- 不复权的话，均线可能因为历史除权跳空而失真

## 6. 实时行情 vs 历史 K 线

| 维度 | 实时行情 | 历史 K 线 |
|------|---------|---------|
| 更新频率 | 秒级（交易时间内） | 每日收盘后更新 |
| 数据量 | 最新一条记录 | 若干月/年的日线数据 |
| 用途 | 判断当前是否触发信号 | 计算均线、回测 |
| 数据源 | 腾讯财经实时接口 | 腾讯/新浪历史K线接口 |

### myQuant 的使用方式

**交易时间内查询行情：**
1. 先拉取最近 N 个月的历史 K 线（用于计算 MA250）
2. 再拉取实时价格
3. 用实时价格**替换历史 K 线最后一行的收盘价**，然后计算指标

```python
# 来自 analyzer.py
close = df['收盘'].copy()
close.iloc[-1] = current_price  # 用实时价替换最后一条，盘中更准确
ma250 = close.rolling(250).mean().iloc[-1]
```

**为什么这样做？**

因为 MA250 需要过去 250 天的收盘价。如果今天还没收盘，最后一天的收盘价应该用"当前价"来近似，这样能更准确地反映盘中状态。

## 7. 数据缓存策略

### 7.1 为什么需要缓存？

如果每次计算指标都去网上实时拉取 250 条 K 线，会有：
- **延迟高**：每次请求耗时 0.5~2 秒
- **频率限制**：监控多个标的时可能被封 IP
- **重复下载**：同一标的的历史数据大部分是一样的

### 7.2 HistoryCache 的设计

`cache.py` 中的 `HistoryCache` 模块提供了内存缓存：

```
第一次请求 600519 的历史K线：
  → 从网络获取，存入内存缓存（带时间戳）

第二次请求 600519 的历史K线（5秒后）：
  → 缓存未过期 → 直接返回缓存，不再请求网络

第三次请求 600519 的历史K线（1小时后）：
  → 缓存已过期 → 重新从网络获取，更新缓存
```

### 7.3 缓存时间设计原则

- 历史 K 线数据：每天只需更新一次（收盘后），缓存可以设较长（如 1 小时）
- 实时行情数据：需要实时性，不缓存（或缓存 15~30 秒）

### 7.4 报警去重（AlertStateManager）

`cache.py` 中的 `AlertStateManager` 负责防止重复报警：

```
第一次触发买入信号（600519，BUY）：
  → 检查今日是否已经报过这个方向的警报
  → 没有 → 发送邮件，记录"今日已报警"

同一天再次触发买入信号（600519，BUY）：
  → 检查今日是否已经报过
  → 是 → 不发邮件（防止邮件轰炸）

第二天再次触发：
  → 日期变了，重置状态
  → 允许发送
```
